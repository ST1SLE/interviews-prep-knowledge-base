## Q4. "Объясни Central Limit Theorem"

> "**CLT**: если взять $n$ независимых одинаково распределённых случайных величин, то их среднее при $n \to \infty$ стремится к нормальному распределению, **независимо от исходного распределения**. Формально: $\bar{X} \sim \mathcal{N}(\mu, \sigma^2 / n)$.
>
> Почему важно:
> 1. Обосновывает A/B тесты — среднее метрики по выборке $\approx$ нормальное → можем применять t-test
> 2. Объясняет, почему нормальное распределение повсюду
> 3. Чем больше $n$, тем точнее оценка среднего ($\sigma / \sqrt{n}$ уменьшается)
>
> Практика: при $n \geq 30$ CLT обычно работает хорошо."

## Q6. "Что такое гипотеза? Как проверять?"

> "**Hypothesis testing** — формализованный способ проверить утверждение по данным.
>
> 1. $H_0$ **(null hypothesis)** — 'ничего не изменилось' (модели одинаковы, кнопка не влияет на конверсию)
> 2. $H_1$ **(alternative)** — 'есть эффект'
> 3. Выбираем тест (t-test, Mann-Whitney, chi-square)
> 4. Считаем **p-value** — вероятность получить такие или более экстремальные данные, если $H_0$ верна
> 5. Если p-value $< \alpha$ (обычно 0.05) → отвергаем $H_0$
>
> **Type I error** (false positive): отвергли $H_0$, когда она верна ($\alpha$)
> **Type II error** (false negative): не отвергли $H_0$, когда она ложна ($\beta$)
> **Power** $= 1 - \beta$: вероятность обнаружить реальный эффект"

## Q7. "Какой тест выбрать?"

| Ситуация | Тест | Когда |
|----------|------|-------|
| Сравнить средние двух групп | **t-test** (Student) | Нормальные данные, непрерывная метрика |
| Сравнить средние, не нормальные данные | **Mann-Whitney U** | Не параметрический аналог t-test |
| Сравнить пропорции (конверсии) | **z-test для пропорций** | A/B тест на CTR |
| Сравнить распределения | **Kolmogorov-Smirnov** | Data drift detection в ML |
| Категориальные переменные | **Chi-square** ($\chi^2$) | Связь между категориями |
| До/после на одной группе | **Paired t-test** | Один и тот же клиент до и после |

> "На собеседовании чаще всего спрашивают про A/B тесты → t-test или z-test для пропорций."

## Q8. "Confidence interval — что это?"

> "**Доверительный интервал** — диапазон, в который попадает истинное значение параметра с заданной вероятностью (обычно 95%).
>
> Для среднего: $CI = \bar{X} \pm z \cdot \frac{\sigma}{\sqrt{n}}$, где $z = 1.96$ для 95%.
>
> Интуиция: если повторить эксперимент 100 раз, в ~95 случаях CI накроет истинное среднее. Чем больше $n$, тем уже CI (точнее оценка). Чем больше дисперсия, тем шире CI.
>
> В ML: можно построить CI для метрик модели через bootstrap — обучаем на K случайных подвыборках, считаем метрику на каждой, берём 2.5% и 97.5% квантили."

## Q15. "Что такое bootstrap?"

> "**Bootstrap** — метод оценки статистик через resampling. Из исходной выборки $n$ элементов создаём $B$ новых выборок размера $n$ **с возвратом** (с повторениями). Для каждой считаем нужную статистику (среднее, медиана, ROC-AUC). Получаем распределение статистики → можем построить CI.
>
> Зачем в ML:
> - Оценить CI для метрик модели без формул (не нужно знать распределение)
> - Random Forest = bootstrap на стероидах: каждое дерево обучается на bootstrap-подвыборке
> - Проверить стабильность feature importance"

```python
import numpy as np

scores = []
for _ in range(1000):
    idx = np.random.choice(len(y_test), size=len(y_test), replace=True)
    score = roc_auc_score(y_test[idx], y_pred[idx])
    scores.append(score)

ci_lower, ci_upper = np.percentile(scores, [2.5, 97.5])
print(f"ROC-AUC: {np.mean(scores):.3f} [{ci_lower:.3f}, {ci_upper:.3f}]")
```

## Q16. "Ошибки I и II рода, мощность теста"

> "**Type I Error** (False Positive, $\alpha$): отвергаем верную $H_0$. Контроль: уровень значимости $\alpha$ (обычно 0.05). Пример в ML: запустили вредную фичу, потому что A/B тест показал ложный 'значимый' результат.
>
> **Type II Error** (False Negative, $\beta$): не отвергаем $H_0$, хотя она ложна. Пропускаем реальный эффект. Пример: полезная фича не прошла A/B тест из-за малого sample size.
>
> **Мощность теста** (Power) $= 1 - \beta$ — вероятность обнаружить реальный эффект, если он существует. Стандарт: power = 0.8 (80%).
>
> Факторы, влияющие на мощность:
> - $n$ (размер выборки) ↑ → мощность ↑
> - Effect size ↑ → мощность ↑
> - $\alpha$ ↑ → мощность ↑ (но больше ложных срабатываний)
> - Дисперсия ↓ → мощность ↑
>
> Компромисс: увеличение $\alpha$ даёт больше мощности, но больше False Positives. На практике: фиксируем $\alpha = 0.05$, подбираем $n$ для нужной мощности."

| Ошибка | Название | Вероятность | ML-пример | Контроль |
|--------|----------|-------------|-----------|----------|
| **Type I** | False Positive | $\alpha$ | Запуск вредной фичи | Уровень значимости $\alpha$ |
| **Type II** | False Negative | $\beta$ | Пропуск полезной фичи | Мощность $1 - \beta$, увеличение $n$ |

```python
from scipy.stats import norm
import numpy as np

def sample_size_for_ab_test(
    baseline_rate: float,
    mde: float,
    alpha: float = 0.05,
    power: float = 0.8,
) -> int:
    """Размер выборки для A/B теста (z-test для пропорций)."""
    z_alpha = norm.ppf(1 - alpha / 2)
    z_beta = norm.ppf(power)
    p1 = baseline_rate
    p2 = baseline_rate + mde
    n = ((z_alpha + z_beta) ** 2 * (p1 * (1 - p1) + p2 * (1 - p2))) / mde ** 2
    return int(np.ceil(n))

n = sample_size_for_ab_test(baseline_rate=0.05, mde=0.01)
print(f"Нужно {n} пользователей на группу")  # ~7000-8000
```

## Q17. "Multiple Hypothesis Testing — поправка Бонферрони"

> "**Проблема множественных сравнений**: при тестировании $n$ гипотез с $\alpha = 0.05$ вероятность хотя бы одного ложного срабатывания:
>
> $$P(\text{хотя бы 1 FP}) = 1 - (1 - \alpha)^n$$
>
> При $n = 20$ метриках: $1 - 0.95^{20} \approx 0.64$ — больше половины!
>
> **Коррекция Бонферрони**: используем $\alpha' = \alpha / n$. При 20 тестах: $\alpha' = 0.05/20 = 0.0025$. Плюс: простая, гарантирует FWER (Family-Wise Error Rate). Минус: слишком консервативная — теряем мощность.
>
> **Benjamini-Hochberg (FDR)**: контролирует **долю** ложных открытий (False Discovery Rate), а не вероятность хотя бы одного. Менее консервативная → больше мощности. Алгоритм: сортируем p-values, сравниваем с порогом $\alpha \cdot k/n$.
>
> Когда Бонферрони: критичные решения (медицина, финансы). Когда BH: exploratory analysis, много метрик в A/B тесте."

| Метод | Контролирует | Консервативность | Когда |
|-------|-------------|-----------------|-------|
| Без коррекции | Ничего | Нет | 1 тест |
| **Бонферрони** | FWER ($P \geq 1$ FP) | **Высокая** | Критичные решения |
| **Benjamini-Hochberg** | FDR (доля FP) | Умеренная | Exploratory, A/B с 10+ метриками |

```python
from scipy.stats import false_discovery_control
import numpy as np

p_values = [0.001, 0.008, 0.039, 0.041, 0.049,
            0.12, 0.15, 0.23, 0.34, 0.45,
            0.51, 0.55, 0.62, 0.71, 0.73,
            0.78, 0.83, 0.88, 0.91, 0.97]

alpha = 0.05
n = len(p_values)

# Бонферрони: alpha/n
bonferroni_significant = [p < alpha / n for p in p_values]
print(f"Bonferroni: {sum(bonferroni_significant)} значимых")  # 2

# Benjamini-Hochberg
sorted_p = sorted(enumerate(p_values), key=lambda x: x[1])
bh_significant = [False] * n
for k, (idx, p) in enumerate(sorted_p, 1):
    if p <= alpha * k / n:
        bh_significant[idx] = True
print(f"BH (FDR): {sum(bh_significant)} значимых")  # 4-5
```
