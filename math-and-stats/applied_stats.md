## Q9. "Correlation vs Causation"

> "**Корреляция** — линейная связь между двумя переменными. Pearson $r \in [-1, 1]$. Correlation $\neq$ causation: продажи мороженого коррелируют с числом утоплений, но мороженое не убивает людей — обе переменные вызваны жарой (конфаундер).
>
> Чтобы доказать causation нужен:
> 1. **Рандомизированный эксперимент** (A/B тест) — устраняем конфаундеры
> 2. **Causal inference** (Pearl's do-operator, backdoor criterion) — когда эксперимент невозможен
>
> В ML это критично: модель находит корреляции, но для принятия решений нужна каузальность. Пример: модель скоринга может найти, что 'цвет телефона' коррелирует с дефолтом — но это не причина, а proxy."

## Q10. "Что такое A/B тест? Расскажи полный pipeline"

> "A/B тест — controlled experiment для проверки гипотезы.
>
> **Pipeline:**
> 1. **Гипотеза**: 'Новый промпт для голосового агента увеличит конверсию' ($H_1$)
> 2. **Метрика**: primary (конверсия), guardrail (time-to-resolution, NPS)
> 3. **Размер выборки**: power analysis — нужно $N$ пользователей для обнаружения MDE (minimum detectable effect) с power=0.8
> 4. **Рандомизация**: случайно делим пользователей 50/50 (или 90/10 для рискованных)
> 5. **Запуск**: собираем данные минимум 1-2 недели (учитываем день недели, сезонность)
> 6. **Анализ**: t-test/z-test, p-value < 0.05 → статистически значимо
> 7. **Решение**: если значимо И практически значимо (effect size достаточный) → раскатываем
>
> **Ошибки:** peeking (смотреть p-value до набора выборки), multiple testing (много метрик без коррекции Bonferroni), selection bias, novelty effect."

## Q11. "Что такое MLE и MAP?"

> "**MLE (Maximum Likelihood Estimation)** — находим параметры $\theta$, которые максимизируют вероятность наблюдённых данных: $\theta_{\text{MLE}} = \arg\max P(\text{data} \mid \theta)$. Не учитывает prior knowledge.
>
> **MAP (Maximum A Posteriori)** — добавляем prior: $\theta_{\text{MAP}} = \arg\max P(\theta \mid \text{data}) = \arg\max P(\text{data} \mid \theta) \cdot P(\theta)$. Эквивалентно MLE + regularization: если prior — Gaussian → L2 regularization, если Laplace → L1.
>
> Пример: оценка вероятности дефолта. MLE на 10 наблюдениях: 3 дефолта → $P = 0.3$. MAP с prior 'обычно дефолтов ~5%' → оценка 'подтянется' к 0.05, особенно при малом $n$. При большом $n$ — MLE и MAP сходятся."

## Q12. "Bias-Variance Tradeoff через призму статистики"

> "Ожидаемая ошибка модели $= \text{Bias}^2 + \text{Variance} + \text{Irreducible Noise}$.
>
> **Bias** — систематическая ошибка, модель слишком простая (underfitting). Высокий bias: линейная регрессия на нелинейных данных.
>
> **Variance** — разброс предсказаний при разных обучающих выборках. Высокая variance: дерево глубины 100 (overfitting) — чуть другие данные → совсем другие предсказания.
>
> **Tradeoff**: увеличиваем сложность → bias ↓ variance ↑. Оптимум — в точке минимальной суммарной ошибки.
>
> **Как бороться с high variance:** regularization, ensemble (RF усредняет деревья → variance ↓), больше данных, dropout.
> **Как бороться с high bias:** более сложная модель, feature engineering, убрать regularization."

## Q13. "Сетевые эффекты (Network Effects) в A/B тестах"

> "**Проблема**: в маркетплейсах, такси, доставке treatment группа влияет на control. Пример Яндекс Такси: если treatment-водителям платить больше → они забирают заказы у control-водителей → control-метрики падают → переоцениваем эффект.
>
> Нарушается **SUTVA** (Stable Unit Treatment Value Assumption) — ключевое допущение A/B тестов, что treatment одного юнита не влияет на outcome другого.
>
> **Решения:**
>
> 1. **Switchback-эксперименты** — чередуем treatment/control во времени (например, час A → час B). Все пользователи в один момент видят одну версию → нет перехвата. Минус: carry-over effects.
>
> 2. **Гео-стратификация** — рандомизация по городам/районам. Район A = treatment, район B = control. Пользователи внутри района не взаимодействуют с другим районом.
>
> 3. **Кластерная рандомизация** — группируем связанных пользователей в кластеры, рандомизируем кластеры, а не юнитов. Эффективный sample size меньше.
>
> Когда встречается: Яндекс Такси (перехват заказов), Яндекс Лавка (общие курьеры), Ozon Fresh (доставка), любой двусторонний маркетплейс."

| Метод | Принцип | Плюсы | Минусы |
|-------|---------|-------|--------|
| **Switchback** | Чередование во времени | Нет перехвата | Carry-over effects, сложный анализ |
| **Гео-стратификация** | Рандомизация по регионам | Изолирует группы | Мало кластеров → низкая мощность |
| **Кластерная рандомизация** | Рандомизация групп связанных юнитов | Учитывает сеть | Эффективный $n$ < реального $n$ |

## Q14. "Оценка размера выборки методом Монте-Карло"

> "Помимо аналитических формул (z-test), размер выборки можно оценить через Монте-Карло симуляцию. Идея: генерируем $K$ выборок размера $n$, считаем среднее отклонение от истинного значения, ищем минимальное $n$, при котором отклонение ≤ допустимого $\delta$. Этот подход универсален — работает для любых статистик, не только среднего."

$$F(n) = \frac{1}{K}\sum_{j=1}^{K} |M_{nj} - a|, \quad \text{ищем наименьшее } n: F(n) \leq \delta$$

```python
K = 500          # число симуляций
n_max = 5000     # максимальный размер выборки
delta = 0.02     # допустимое отклонение
a = 0.5          # истинное среднее

# Генерация K выборок длины n_max
rng = np.random.default_rng(42)
samples = rng.binomial(1, a, size=(K, n_max))

# Кумулятивные средние для всех n (без циклов — векторизация!)
cumsum = np.cumsum(samples, axis=1)
ns = np.arange(1, n_max + 1)
running_means = cumsum / ns                     # shape (K, n_max)
F_n = np.mean(np.abs(running_means - a), axis=0)  # shape (n_max,)

# Наименьшее n с F(n) ≤ δ
n_star = np.argmax(F_n <= delta) + 1
print(f"Необходимый размер выборки: {n_star}")
```

**Теоретическая оценка (Бернулли):** $n \geq 2a(1-a) / (\pi \delta^2)$.

**Gotcha:** если данные коррелированы (временные ряды, сессии), реальный размер выборки может быть значительно больше — см. эффективный размер выборки ниже.

## Q15. "Марковские цепи и эффективный размер выборки"

> "Марковская цепь — последовательность, где каждый элемент зависит только от предыдущего (свойство 'без памяти'). Ключевое понятие для A/B тестов и MCMC: при автокорреляции в данных эффективный размер выборки меньше реального."

### Эффективный размер выборки

При автокорреляции $\rho = 2p - 1$ (для двухсостояной цепи с вероятностью 'остаться' $p$):

$$n_{\text{eff}} = \frac{n(1-\rho)}{1+\rho} = \frac{n(1-p)}{p}$$

| Ситуация | $\rho$ | $n_{\text{eff}}$ | Следствие |
|----------|--------|-------------------|-----------|
| Положительная корреляция | > 0 | < $n$ | Нужно больше данных |
| Отрицательная корреляция | < 0 | > $n$ | Чередование ускоряет сходимость |
| Независимые наблюдения | = 0 | = $n$ | Стандартный случай |

> **Практика:** в MCMC, A/B тестах с сессионной корреляцией, и временных рядах всегда проверяй автокорреляцию перед оценкой размера выборки. Иначе CI будут слишком узкими, а p-values — оптимистичными.

### Векторизованная симуляция Марковской цепи

```python
# K независимых цепей длины n_max — без циклов
p = 0.8  # вероятность остаться в текущем состоянии
U = rng.uniform(size=(K, n_max))
switches = (U >= p).astype(int)              # 1 = переключение
states = np.cumsum(switches, axis=1) % 2     # кумулятивный XOR
```
