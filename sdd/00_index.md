# Spec-Driven Development (SDD) with OpenSpec

Study materials for learning SDD from scratch using the [OpenSpec](https://github.com/Fission-AI/OpenSpec) framework.

---

## Learning Path

| # | Module | Focus | Time |
|---|--------|-------|------|
| 1 | [01_fundamentals.md](01_fundamentals.md) | Why SDD exists, alignment levels, unified vs fragmented topologies | Theory |
| 2 | [02_setup.md](02_setup.md) | Install OpenSpec, `openspec init`, AGENTS.md context checklist | Hands-on |
| 3 | [03_core_lifecycle.md](03_core_lifecycle.md) | Propose → Apply → Archive — the core workflow with exercises | Hands-on |
| 4 | [04_brownfield.md](04_brownfield.md) | Applying SDD to legacy codebases — Explore → Continue | Hands-on |
| 5 | [05_custom_schemas.md](05_custom_schemas.md) | Custom artifact pipelines — minimalist, EDA, domain-specific | Advanced |
| 6 | [06_orchestration.md](06_orchestration.md) | Git WorkTrees, CI/CD gates, MCP integration, contract testing | Advanced |

**Quick reference:** [cheatsheet.md](cheatsheet.md)

---

## How to Use

1. Read modules **in order** — each builds on the previous
2. Do the exercises at the end of each module
3. Use [cheatsheet.md](cheatsheet.md) as a daily reference once you're comfortable with the workflow
