# Основы AI-агентов — Interview Prep

Agent vs LLM, ReAct, память агента, CoT vs Tool Use для собеседования ML/DS/AI Engineer Intern.

---

## Q1. "Что такое AI-агент? Чем отличается от простого LLM?"

> "Обычный LLM — это 'один запрос → один ответ'. AI-агент — это LLM, который умеет **рассуждать, принимать решения и действовать**: вызывать инструменты, обращаться к базам данных, взаимодействовать с API, и делать это итеративно, пока задача не решена.
>
> Ключевые компоненты агента:
> 1. **LLM** — 'мозг' (рассуждение и планирование)
> 2. **Tools** — инструменты (поиск, калькулятор, API, базы данных)
> 3. **Memory** — контекст (короткая: chat history, длинная: vector DB)
> 4. **Planning** — стратегия решения задачи (разбиение на подзадачи)
>
> Аналогия: LLM — это мозг в банке, агент — это робот с руками и глазами."

---

## Q2. "Объясни ReAct паттерн"

> "**ReAct** (Reasoning + Acting) — основной паттерн для агентов. На каждом шаге LLM:
> 1. **Thought**: рассуждает — что нужно сделать?
> 2. **Action**: вызывает инструмент с параметрами
> 3. **Observation**: получает результат от инструмента
> 4. Повторяет до решения задачи
>
> Пример: 'Какая погода в Москве и нужен ли зонт?'
> - Thought: нужно узнать погоду в Москве
> - Action: weather_api(city='Moscow')
> - Observation: temperature=5°C, rain_probability=80%
> - Thought: вероятность дождя высокая, нужен зонт
> - Answer: В Москве 5°C, вероятность дождя 80% — возьмите зонт.
>
> Альтернативы: Plan-and-Execute (сначала планируем все шаги, потом выполняем), Reflexion (агент анализирует свои ошибки и исправляет)."

## Q4. "Какие типы памяти у агента?"

> "1. **Short-term memory** — история текущего диалога (chat messages). Ограничена context window (128K у GPT-4, 200K у Claude). Для длинных сессий нужно summarization или sliding window.
>
> 2. **Long-term memory** — vector database (Chroma, Pinecone, pgvector). Агент сохраняет важные факты, предпочтения пользователя, прошлые решения. При новом запросе делает similarity search и подгружает релевантное.
>
> 3. **Working memory** — scratchpad для текущей задачи: промежуточные результаты, план действий, состояние выполнения.
>
> В контексте X5: голосовой агент должен помнить контекст звонка (short-term) и историю обращений клиента (long-term из CRM)."

## Q14. "Chain-of-Thought vs Tool Use — когда что?"

> "**Chain-of-Thought (CoT)** — LLM рассуждает пошагово в тексте. Хорошо для логических задач, математики, рассуждений. Не нужен доступ к внешним данным.
>
> **Tool Use** — LLM вызывает внешние инструменты. Нужен, когда ответ зависит от реальных данных (текущая цена, статус заказа, содержимое БД).
>
> Агент комбинирует оба: CoT для рассуждений ('мне нужно проверить заказ'), Tool Use для действий ('вызываю check_order'). ReAct = CoT + Tool Use в цикле.
>
> Пример: 'Посчитай стоимость доставки из Москвы в Казань для 5 кг'
> - CoT: нужно знать тариф → нужно вызвать API
> - Tool: `get_delivery_rate(from='Moscow', to='Kazan', weight=5)`
> - CoT: тариф 450₽, + НДС 20% = 540₽
> - Answer: 540₽"
